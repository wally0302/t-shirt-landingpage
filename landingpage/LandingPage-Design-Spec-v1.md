## 1. 參考現有網站內容（盤點）
本文件以目前程式與文件內容為基礎：
- 品牌與語氣：`Wardrobe / 衣櫥`、中英混合語系。
- 目前首頁：`/` 直接導向 `/login`（尚無對外宣傳頁）。
- 目前核心功能：
  - Google 登入
  - 上傳衣物照片（JPG/PNG/WEBP）
  - 自動壓縮為 thumb/view
  - 搜尋 / 分類 / 排序 / 編輯 / 刪除
  - 每日上傳配額（10/day，台北時區日切）
  - 天氣卡（定位 / 城市搜尋 / 溫度體感降雨風速）
- 主要場景：快速記錄衣物、出門前依天氣查找穿搭單品。

## 2. Landing Page 目標
1. 將「工具型衣櫃 App」轉為可理解的產品價值敘事。
2. 明確導向轉換行為：
   - 主轉換：點擊「開始整理衣櫥」進入登入/使用流程。
   - 次轉換：訂閱更新（為未來部落格/內容經營預熱）。
3. 保留未來擴充空間：從單頁宣傳，延伸到「穿搭指南 / 產品部落格」。

## 3. 目標受眾
- Persona A：常整理穿搭的使用者（重視分類、搜尋效率）。
- Persona B：出門前快速找衣服者（重視天氣 + 快速定位單品）。
- Persona C：內容導向訪客（先看穿搭文章，再轉為產品使用者）。

## 4. 核心價值主張（Value Proposition）
- 一句話版本：
  - 「把你的衣櫥變成可搜尋、可管理、可搭配天氣的數位空間。」
- 三個支柱：
  - 快速上傳：照片即整理，不再散落相簿。
  - 快速找到：搜尋/分類/排序，出門前不用翻很久。
  - 情境輔助：在衣櫃頁就能看天氣，降低切換成本。

## 6. 單頁區塊規格（Landing Page）
### 6.1 Hero（首屏）
- 目標：3 秒內講清楚產品做什麼。
- 內容元素：
  - H1：中文主標 + 英文輔助（延續現有品牌調性）。
  - 簡短副標：聚焦「整理衣櫥 + 快速找到 + 天氣輔助」。
  - Primary CTA：開始整理衣櫥（連到 `/login`）。
  - Secondary CTA：看功能亮點（頁內錨點）。

### 6.2 Problem → Solution
- 問題（左）：照片分散、找衣困難、出門前資訊斷裂。
- 解法（右）：上傳即分類、搜尋篩選、同頁看天氣。

### 6.3 Feature Highlights（3~4 張卡）
- 功能卡建議：
  - 上傳與壓縮：支援常見格式、流程輕量。
  - 管理與搜尋：分類/顏色/備註檢索。
  - 天氣卡整合：定位或城市切換。
  - 額度與節奏：每日配額機制，建立整理習慣。

### 6.4 How It Works（3 步）
- 1. 登入
- 2. 新增衣物
- 3. 依天氣與搜尋快速找到要穿的單品

### 6.5 Future Content Preview（為部落格鋪路）
- 區塊名稱：穿搭指南 / 最新文章（可先放 3 張預告卡）。
- 目的：讓 landing page 從「功能頁」轉為「內容入口」。
- 初期可用假資料卡（不需後端）。

### 6.6 FAQ
- 建議問題：
  - 現在支援什麼登入方式？
  - 上傳格式有哪些？
  - 每日上傳限制如何計算？
  - 未來會有穿搭指南嗎？

### 6.7 Final CTA + Footer
- Final CTA：再次導向 `/login`。
- Footer：品牌名、導覽、聯絡方式、版權與隱私連結（可先占位）。

## 7. 內容與文案風格
- 語言策略：繁中為主，英文為輔（與現有 UI 一致）。
- 語氣：清楚、輕量、實用，不過度行銷詞。
- 文案結構：
  - 標題講結果（例如「30 秒內新增一件衣物」）
  - 內文講機制（上傳、分類、天氣整合）

## 8. 視覺設計方向（延續現有風格）
- 色彩：延用既有 token 的淺底 + 紫色主色（primary / soft）。
- 元件語言：圓角卡片、細邊框、輕陰影、柔和互動動效。
- 圖像建議：
  - 產品截圖（衣物卡片網格、新增 modal、天氣卡）
  - 情境插圖（出門前快速找衣）
- 響應式：
  - Desktop：完整多欄
  - Tablet：區塊重排
  - Mobile：單欄與大型 CTA 優先

## 9. SEO 與內容可發現性
- Metadata：
  - Title：Wardrobe / 衣櫥｜你的數位穿搭管理工具
  - Description：管理衣物、快速搜尋、結合天氣資訊的線上衣櫃。
- Open Graph：首頁主視覺 + 產品一句話價值。
- URL 原則：語意化、短路徑（`/blog/how-to-organize-your-closet`）。
- 部落格上線後：
  - 每篇文章唯一 title/description
  - 統一 heading 階層與內部連結

## 10. 量測事件（Landing 版）

## 11. 第一版驗收標準（Design 層）
1. 首頁首屏可明確說明產品價值與主要 CTA。
2. 使用者能在不登入情況下理解核心功能與使用流程。
3. 頁面具備「未來內容化」的版位（文章預告區）。
4. 手機版可完整閱讀，CTA 可直接點擊操作。
